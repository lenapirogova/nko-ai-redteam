# nko-ai-redteam

**Red teaming инструмент для проверки раздела «Обоснование социальной значимости» в грантовых заявках НКО**

---

## Зачем это нужно

ИИ уверенно генерирует статистику, названия исследований и ссылки на нормативные акты, которые либо не существуют, либо содержат неточные данные. В разделе социальной значимости это особенно опасно: именно этот раздел эксперты грантодателя проверяют особенно внимательно.

Инструмент создан по образцу того, как юристы решают похожую проблему — они строят adversarial-верификацию, которая систематически «атакует» AI-контент перед тем, как он уйдёт к клиенту.

## Что внутри

`SKILL_grant_redteam_ru.md` — инструкция для Claude (или любого другого ИИ), которая превращает его в adversarial-верификатор. Включает:

- **Иерархию источников** под российский контекст (Росстат, Минздрав, КиберЛенинка, база ФПГ и др.)
- **6 типичных паттернов галлюцинаций** в грантовых заявках — от несуществующих исследований до региональных данных, поданных как федеральные
- **Пошаговую методологию проверки** — от первичного обзора до оценки непроверяемых утверждений
- **Шаблон отчёта** с уровнями серьёзности ошибок (критические / высокие / средние / низкие)
- **Руководство по базе проектов-победителей ФПГ** — как использовать открытые данные фонда для дополнительной верификации

## Как использовать

Никакого кода и технической подготовки не нужно. Достаточно Claude или ChatGPT.

1. Откройте `SKILL_grant_redteam_ru.md`
2. Скопируйте содержимое в начало диалога с ИИ (или загрузите файл)
3. Вставьте текст раздела социальной значимости из вашей заявки
4. Попросите: *«Проверь этот раздел по инструкции»*
5. Получите структурированный отчёт с конкретными ошибками и рекомендациями

## Для каких грантов

В первую очередь для **Фонда президентских грантов** — под него заточены источники и паттерны. Логика верификации применима к другим российским грантодателям: Фонд Тимченко, Фонд Потанина, региональные конкурсы.

## Как помочь доработать

Инструмент создан одним человеком и намеренно сделан без кода — чтобы в доработке мог участвовать любой практик из сектора, а не только технари.

Буду рада вкладу от коллег, которые пишут заявки в конкретных тематических направлениях: здоровье, инвалидность, образование, социальное обслуживание и др. Каждое направление имеет свою специфику источников и типичные ошибки.

Форматы участия:
- **Issues** — сообщить об ошибке, предложить дополнение, описать паттерн галлюцинации, которого нет в файле
- **Pull requests** — предложить правки напрямую
- **Просто написать** — если с GitHub незнакомы, можно написать напрямую в Телеграм: [@mrsbroccoli](https://t.me/mrsbroccoli)
